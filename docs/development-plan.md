# 开发执行步骤

## 总体时间线
- 总工期：约 10 周（50 个工作日）
- 开发方式：分 4 个阶段，每个阶段有明确的交付物
- 原则：循序渐进，每步验证后再进入下一步

---

## 第一阶段：基础骨架 + 数据采集（约 3 周）

### 1.1 项目初始化（第 1-2 天）
- [ ] 初始化 Node.js 项目（package.json）
- [ ] 安装 Electron + React + TypeScript + Vite 依赖
- [ ] 配置 tsconfig.json
- [ ] 配置 vite.config.ts（主进程 + 渲染进程双构建）
- [ ] 配置 electron-builder.yml
- [ ] 创建基础目录结构（src/main, src/preload, src/renderer）
- [ ] 验证：能启动 Electron 窗口并显示 React 页面

### 1.2 数据库搭建（第 3-4 天）
- [ ] 安装 better-sqlite3
- [ ] 编写数据库初始化代码
- [ ] 编写 Schema 迁移系统（版本化管理）
- [ ] 创建五张核心表（districts, projects, buildings, units, price_snapshots）
- [ ] 创建 sync_log 和 app_settings 表
- [ ] 编写基础 CRUD 查询封装
- [ ] 验证：数据库文件创建成功，能写入和读取测试数据

### 1.3 网站接口侦查（第 5-7 天）
- [ ] 在浏览器中打开 https://fsfc.fszj.foshan.gov.cn/loupan/s/
- [ ] 按 F12 打开 Network 面板
- [ ] 切换五个区域，记录所有 XHR/Fetch 请求
- [ ] 点击具体楼盘，记录楼栋和房源数据请求
- [ ] 记录每个接口的：URL、方法、参数、响应 JSON 结构
- [ ] 确认是否需要 Cookie/Token
- [ ] 确认分页机制
- [ ] 输出：接口文档（保存到 docs/api-reference.md）

### 1.4 数据采集模块（第 8-12 天）
- [ ] 实现 rate-limiter.ts（请求频率控制）
- [ ] 实现 api-client.ts（基于 axios 的 API 调用封装）
- [ ] 实现 page-scraper.ts（Electron BrowserWindow 页面抓取）
- [ ] 实现 parser.ts（数据清洗、标准化、9折计算）
- [ ] 实现采集编排器（遍历五区→项目→楼栋→房源→价格）
- [ ] 实现增量采集逻辑（对比已有数据）
- [ ] 实现断点续采
- [ ] 实现采集进度通知
- [ ] 验证：能成功抓取一个区域的真实数据并存入数据库

### 1.5 基础 UI（第 13-15 天）
- [ ] 实现 Electron 主窗口（带侧边栏布局）
- [ ] 实现 Sidebar 组件（区域列表 + 搜索框入口）
- [ ] 实现 Dashboard 页面（五区卡片总览）
- [ ] 实现 DistrictView 页面（楼盘列表，单价降序）
- [ ] 实现 ProjectTable 组件
- [ ] 实现 ScrapeButton 组件（手动触发采集 + 进度显示）
- [ ] IPC 通道配置（主进程与渲染进程通信）
- [ ] 验证：点击刷新→采集数据→五区卡片→点击区域→看到楼盘列表

## 每日工作流程

1. 查看 [开发日志](../devlog/) 中的待办事项
2. 完成当天的开发任务
3. 更新开发日志：已完成事项 + 待办事项 + 遇到的问题
4. 每完成一个子任务，勾选上方对应复选框
